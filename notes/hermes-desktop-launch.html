先说背景

Hermes Agent 是 Nous Research 今年 2 月推出的开源 AI agent 框架。上期 Digest（#006）介绍过它的核心理念：重点不是聊天，是记忆。它和 ChatGPT、Claude 最大的区别是，它跨会话记住你怎么做事，会把解决问题的方法沉淀成可复用的"技能"，而且常驻在你的本地机器上。

问题是，之前只能通过终端命令行使用。对大量非技术用户来说，这基本等于"看得见、用不了"。社区陆续做了一些第三方 GUI，但都不是官方出品，体验参差不齐。

6 月 2 日，Nous Research 自己动手了。

Hermes Desktop 是什么

Hermes Desktop v0.15.2 是官方出品的原生桌面应用，不是社区做的。Decrypt 的标题是 "Hermes Ends AI Agent Terminal Era"。

几个关键点：

  • 和 CLI 共享同一个 agent 核心：记忆、技能、配置、API keys、会话历史全部通用。桌面版只是一个新的交互界面，不是一个分叉产品
  • 一键安装 + 应用内自动更新：不需要装 Python、不需要配环境变量、不需要命令行
  • 拖拽文件聊天：直接把文件拖进对话窗口
  • 内置模型选择器：状态栏里切换模型（可以用 OpenAI、Anthropic、本地 Ollama 等）
  • 多 Profile 并发：可以同时运行多个 agent 配置
  • Electron + React + Python 后端：技术栈主流，社区好贡献

注意：当前是 Public Preview 阶段，预期会有粗糙的地方。Windows 安装器没有代码签名，SmartScreen 会弹安全警告，需要手动点"仍要运行"。

为什么 Hermes 和 ChatGPT/Claude 桌面版不一样

这三个产品看起来都是"桌面上的 AI 对话窗口"，但定位完全不同：

Hermes Desktop

开源本地 agent

跨会话持久记忆

技能自学 + 沉淀

定时任务 + 后台常驻

多平台同步（Telegram / Discord / Slack / 邮件）

自带模型（可本地可云端）

"住在你机器上的私人 agent"

Claude Desktop

Anthropic 官方客户端

会话内深度推理强

MCP 工具集成

记忆有限（项目级）

绑定 Claude 模型

订阅制

"最强的 AI 工作伙伴"

ChatGPT Desktop

OpenAI 官方客户端

屏幕共享 + 语音对话

插件生态

无跨会话记忆

绑定 GPT 模型

订阅制

"随时能聊的 AI 助手"

简单说：ChatGPT 桌面版是"随叫随到的对话"，Claude 桌面版是"深度工作的搭档"，Hermes 是"越用越懂你的本地 agent"。它们不是互斥的，很多人会同时用不止一个。

Hermes 的核心能力拆解

1. 持久记忆

Hermes 会跨会话记住你说过什么、做过什么、偏好什么。不是"聊天记录搜索"，而是结构化的用户模型。你不需要每次开新对话都重新解释自己是谁、在做什么项目。

2. 技能自学

当 Hermes 帮你解决了一个问题，它会把解决方案沉淀成一个"技能"。下次遇到类似问题，直接调用，不需要重新推理。时间越长，它能做的事越多。

3. 多平台一个记忆

桌面版、CLI、Telegram、Discord、Slack、WhatsApp、Signal、邮件。不管你从哪个入口和它对话，它都是同一个 agent、同一份记忆。这是目前市面上很少有产品做到的。

4. 定时任务 + 后台常驻

你可以让 Hermes 定期检查某个数据源、定时发提醒、按计划执行任务。它不只在你打开窗口时工作。

成本：多少钱

方案月成本配置
纯本地$0（只有电费）Ollama 跑本地模型，适合有 GPU 的机器
经济型~$6-8便宜 VPS + DeepSeek V4 API
高配型$30-80更好的 VPS + Claude Sonnet 4.6 API

Hermes 本身完全免费（MIT 开源）。你付的是底层模型的 API 调用费和（如果需要）服务器费用。如果你的电脑性能够用，跑本地模型可以做到零成本。

怎么上手

最快的路径：

  1. 去 hermes-agent.nousresearch.com/desktop 下载对应系统的安装包
  2. 安装并打开（Windows 会弹 SmartScreen 警告，点"仍要运行"）
  3. 在状态栏选择模型（推荐先用 OpenRouter 或 Anthropic API，体验最好）
  4. 开始对话，它会自动建立你的用户档案和记忆

如果你之前用过 CLI 版：桌面版会自动读取你已有的配置、记忆和技能，无缝衔接。

现阶段的限制

  • Public Preview：不是正式 1.0，会有 bug 和缺失功能
  • 没有 IDE/代码编辑器集成：如果你主要需求是 coding copilot，Claude Code 或 GitHub Copilot 更合适
  • Windows 安装器未签名：SmartScreen 警告可能吓到不熟悉的用户
  • 需要一定的模型配置理解：虽然比 CLI 简单很多，但"选哪个模型、API key 怎么配"对纯小白还是有门槛
  • Telegram 网关 token 成本偏高：通过 Telegram 用的话，token 消耗是 CLI 的 2-3 倍

So What：跟你有什么关系

如果你想要一个"越用越懂你"的 AI 助手：Hermes 目前是这个方向上最成熟的开源方案。桌面版的发布意味着你不再需要会用终端。

如果你已经在用 Claude / ChatGPT：Hermes 不是替代品，而是互补。Claude 在深度推理和编码上更强，ChatGPT 在日常对话和语音上更方便，Hermes 的优势在"长期记忆 + 自动化 + 多平台同步"。可以同时用。

如果你关注 AI 行业趋势：一个开源项目四个月拿到 180K stars，说明市场对"本地优先、记忆持久、用户可控"的 agent 有真实需求。这和大厂的云端 agent 路线形成了有意思的互补。
