用 AI 打造你的「第二大脑」
Apr 2026 · 基于 Tiago Forte《Building a Second Brain》· DMV 职场女性适配版
你的大脑是用来产生想法的，不是用来储存想法的

这本书讲了什么

Tiago Forte 的《Building a Second Brain》2022 年出版，全球卖了几百万册。核心就一句话："Your mind is for having ideas, not holding them."——你的大脑是用来产生想法的，不是用来储存想法的。

他提出了一个方法：用数字工具建一个"第二大脑"——把你读到的、想到的、学到的东西系统性地存起来，需要的时候随时调用。听起来简单，但大多数人的实际状态是：看了很多，记住很少，能用的更少。

原书的方法论有两个核心框架：

CODE — 工作流

Capture · 捕捉 — 遇到有共鸣的信息，随手存下
Organize · 整理 — 按"能用来做什么"分类，不按主题
Distill · 蒸馏 — 提炼关键洞察，让笔记未来的自己看得懂
Express · 表达 — 用你的积累产出东西（文章/方案/决策）

PARA — 分类法

Projects · 项目 — 有明确截止日期的事
Areas · 领域 — 你长期负责的事（健康/财务/育儿）
Resources · 资源 — 你感兴趣的主题（AI/投资/旅行）
Archives · 归档 — 不活跃的项目和资料
💡 为什么这本书现在对你更有用了
2022 年出版时还没有 ChatGPT、Claude、NotebookLM。现在这些 AI 工具可以自动化 CODE 流程的前三步——捕捉、整理、蒸馏都可以让 AI 帮忙做。你只需要专注最后一步：表达。这也是原文说的"AI 时代你最大的竞争力不是记忆力，而是把散落信息组装成洞察的能力。"

手把手搭建：AI 时代的第二大脑

我们把原书的 CODE 流程和 AI 工具结合，分 5 步走。每一步都标注了工具和时间投入。

Step 1 · Capture 捕捉 — 建立信息入口

原书说的是"Keep what resonates"——不要分析，先存下来。凭直觉，觉得有用就收。分析是后面的事。

关键是降低捕捉的摩擦。如果存一条信息要打开三个 App、复制粘贴、选分类，你三天就放弃了。

推荐入口（按场景）
1
浏览网页 / 读文章
Notion Web Clipper（Chrome 扩展）— 一键保存网页到 Notion 指定数据库。自动抓标题、URL、摘要。也可以用 Readwise Reader（$8/月）做高亮标注，自动同步到 Notion。
2
读 PDF / 论文 / 长文
NotebookLM — 丢进去自动出摘要。把摘要复制到 Notion。或者直接用 Claude 上传 PDF 问问题，关键段落存 Notion。
3
听播客 / 看视频
NotebookLM 支持 YouTube 链接，自动转录+摘要。或用 Snipd（播客 App）自动高亮关键段落，导出到 Notion。
4
微信群 / 聊天里看到好东西
截图 → 发给 Claude → "帮我把这张截图的核心要点提炼成 3 条" → 复制到 Notion。最简单的方式。
5
脑子里突然冒出的想法
iPhone 用 Notion 的 Quick Note 小组件；或者直接语音备忘录 → 后面让 Claude 转文字整理。不要让想法在脑子里多停留 30 秒。
💡 原书金句
"In deciding which passages, images, theories, or quotes to keep, don't make it a highly intellectual, analytical decision." — 存东西的时候不要过度分析，凭直觉。分析是 Step 3 的事。

Step 2 · Organize 整理 — 用 PARA 分类，不按主题

这是大多数人做错的地方。你习惯性地建文件夹叫"心理学""育儿""AI 工具"——但这些按主题分的文件夹会越来越大，越来越没用。

PARA 的核心思路：按"能拿来做什么"分类，不按"这是关于什么的"分类。

Notion 数据库实操
PARA 分类定义你的例子
Projects
项目		 有明确终点和截止日期的事 · 准备 Q3 项目 proposal
· 求职（投 10 家目标公司）
· 这周的 AI Digest
Areas
领域		 你长期负责的、需要维持标准的事 · 健康/锻炼
· 孩子教育
· 财务管理
· 职业发展
Resources
资源		 你感兴趣但没有截止日期的主题 · AI 工具合集
· 投资学习
· 旅行灵感
· 菜谱
Archives
归档		 已完成或暂停的项目和资料 · 去年的报税资料
· 上一份工作的笔记
· 完成的课程材料
⚠️ 最常见的错误
看到一篇"如何提高韧性"的文章 → 存进"心理学"文件夹 → 三个月后你要做一个关于团队 resilience 的 presentation → 你完全忘了这篇文章存在。

正确做法：存的时候问自己"这个信息最可能用在我哪个正在做的项目上？"。如果答案是"团队管理"相关的项目 → 存到那个 Project 下面。没有对应项目 → 放到 Resources。

Step 3 · Distill 蒸馏 — 让 AI 帮你提炼

原书提出了 Progressive Summarization（渐进式摘要）——每次回看笔记时再精炼一层。这是原书最费时间的一步，但现在 AI 可以帮你做 80%。

AI 蒸馏工作流
1
第一层：AI 自动摘要
把文章/PDF 丢给 NotebookLM 或 Claude，让它生成 3-5 条核心要点。这是粗筛，60 秒完成。
2
第二层：你标注"跟我有关的"
AI 给的摘要里，用粗体标出跟你当前项目/生活有关的那一两句。这一步必须你自己做——只有你知道什么跟你有关。10 秒完成。
3
第三层：写一句"So What"
用你自己的话写一句：这条信息对我意味着什么？或者我可以怎么用？这就是你的 AI Digest 每条都在做的事——亮点说人话 + So What。20 秒完成。
把这篇文章的核心观点提炼成 3-5 条，每条一句话。 然后告诉我： 1. 对一个在 DMV 做 [你的职业] 的人来说，最值得关注的是哪条？为什么？ 2. 我可以立刻采取的一个行动是什么？
⬆️ 蒸馏用 Prompt · 适用于 Claude / ChatGPT / Gemini
💡 关键区分
AI 可以帮你做摘要和分类，但"这跟我有什么关系"只有你自己能回答。这一步不能省。这也是第二大脑和普通收藏夹的本质区别——收藏夹是"存了就忘"，第二大脑是"存的时候就想好了怎么用"。

Step 4 · Express 表达 — 用你的积累产出东西

原书最重要的观点：笔记不是用来收藏的，是用来用的。你的第二大脑的价值，只在你从中产出东西的时候才体现。

Forte 提出了一个概念叫 Intermediate Packets（中间产物）——把大项目拆成小的、可复用的模块。这些模块存在你的第二大脑里，未来可以组装到新项目中。

哪些算"中间产物"

📝 一份调研摘要
📊 一张对比表格
✉️ 一封写好的邮件模板
📋 一个会议议程框架
🔍 一段文献综述
📐 一份 PPT 大纲

实际用法

上次做的项目 proposal 大纲 → 这次改改就能用
之前整理的 AI 工具对比表 → 直接引用到新 report
三个月前写的邮件模板 → 改几个词发出去
去年的读书笔记 → 拿出来支撑今天的观点

你做的每一期 AI Digest 本身就是"中间产物"——下次有人问你"AI 有什么好用的工具？"你不需要从头想，直接发链接。

Step 5 · 落地：Notion 模板实操

不管你用 Notion、Obsidian 还是 Apple Notes，关键是一个地方存所有东西。这里用 Notion 演示，因为免费版支持 10 人协作、手机端完整可用、不需要学 Markdown。

推荐的 Notion 数据库结构
数据库名用途关键字段
📥 Inbox 所有新捕捉的信息先到这里 Source, Date, Status（未处理/已整理/已归档）
📂 PARA 主数据库，按 PARA 四类分 Type（P/A/R/A）, Tags, Related Items, Status
💡 Insights 蒸馏后的 So What 精华 One-liner, Source Link, Applicable To, Date
🧰 Tools 测试过的工具/方法记录 Name, Use Case, Verdict（推荐/观望/不推荐）, Cost
💡 每周节奏（15 分钟/周）
每天：遇到好内容 → 30 秒存进 Inbox（Web Clipper / 截图 / 语音）
周末花 15 分钟：打开 Inbox → 每条问"这跟我哪个项目有关？" → 移到 PARA 对应分类 → 有洞察就写一句 So What 存进 Insights → 清空 Inbox
原则：Inbox 永远不应该超过 20 条。超过了说明你存太多或整理太少。

AI 时代的升级：原书没有的部分

原书写于 2022 年，AI 工具还不成熟。现在我们可以把 CODE 流程的效率提升 5-10 倍：

CODE 步骤原书方法（手动）2026 AI 加持版
Capture 手动复制粘贴、截图 Web Clipper 一键存 + Claude 提取截图内容
Organize 手动拖拽分类到 PARA 让 Claude 建议"这条信息应该放在哪个 Project/Area 下"
Distill 手动高亮 → 手动写摘要 NotebookLM 自动摘要 → 你只标注 So What
Express 从零开始写 把 Insights 喂给 Claude → 它帮你组装成 draft → 你改
AI 做前三步，你做最后一步
捕捉、整理、蒸馏让 AI 帮忙 · 判断和表达只有你能做

跟你有什么关系

如果你是非技术背景：从 Notion + NotebookLM 开始就够了。每天存 1-2 条，周末整理 15 分钟。一个月后你会发现自己有了一个"随时能查的知识库"——写 report、准备会议、做决策的时候不再从零开始。

如果你是技术背景：可以进一步用 Notion API + Claude API 自动化整理流程。甚至把你的第二大脑变成一个 RAG（检索增强生成）系统——问它问题，它从你的笔记里找答案。

如果你在带团队：PARA 结构特别适合团队知识管理。把团队的 Confluence/Google Drive 按 PARA 重新组织一遍，新人 onboarding 速度能快一倍。

如果你在找工作：把求职信息、公司调研、面试准备都按 PARA 存——"求职"是一个 Project，里面分"目标公司""简历版本""面试复盘"。配合 Issue #002 的求职 Bot，形成闭环。
