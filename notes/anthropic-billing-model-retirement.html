Anthropic 计费改革 + 模型退役：你需要知道的变化

6 月 15 日是两件事同时生效的日子：订阅计费拆分为交互池和 Agent SDK 池，以及 Sonnet 4.0 / Opus 4.0 初版正式退役。先说结论：日常使用几乎不受影响，但 API 用户需要注意迁移。
AI Digest Issue #007 · 生效日期 2026.06.15 · By Syneira · 来源：Anthropic 官方文档

Part 1：计费拆分

6 月 15 日起，Anthropic 会把订阅额度拆成两个独立池：

交互池（不变）

Claude.ai 网页/桌面/手机端、Claude Code 终端交互、Claude Cowork 协作功能。这些用法继续走现有的订阅限额，使用体验不变。

Agent SDK 信用池（新增）

Agent SDK、claude -p（编程模式）、Claude Code GitHub Actions、第三方基于 Agent SDK 构建的应用。这些用法从订阅里独立出来，单独计费。

订阅级别Agent SDK 月度额度计费方式
Pro$20/月按 API 费率消耗，不滚存
Max 5x$100/月同上
Max 20x$200/月同上

操作提醒：6 月 8 日会收到 Anthropic 的 claim email，点一次激活即可，之后每个计费周期自动刷新。如果你不用 Agent SDK 相关功能，这个变化对你零影响。

Part 2：模型退役

同样在 6 月 15 日 9AM PT，以下两个模型正式退役：

模型模型 ID状态
Claude Sonnet 4（初版，2025.05）claude-sonnet-4-20250514退役
Claude Opus 4（初版，2025.05）claude-opus-4-20250514退役

退役后，对这些模型 ID 的 API 调用会返回错误。

不受影响的模型：Claude Opus 4.5、4.6、4.7、4.8（最新）以及 Claude Sonnet 4.5、4.6 全部正常使用。退役的只是 2025 年 5 月的最初版本。如果你用的是近半年内的任何模型版本，不需要做任何事。

如果你是 API 用户

检查你的代码里是否硬编码了 claude-sonnet-4-20250514 或 claude-opus-4-20250514。如果有，建议迁移到：

  • Sonnet 系：claude-sonnet-4-5 或更新版本
  • Opus 系：claude-opus-4-7（推荐用于 agentic 任务）或 claude-opus-4-8（最新旗舰）

So What：一句话总结

如果你只用 Claude.ai 网页版或者 Claude Code 终端对话，这两个变化对你基本无感。需要动手的是：用 Agent SDK 做自动化的开发者（留意新计费池），以及 API 里还在调用初版 4.0 模型的项目（6/15 前迁移）。
