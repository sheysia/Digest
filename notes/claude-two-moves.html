Enterprise

企业那头：用 AWS 账号直接用 Claude

Claude Platform on AWS 今天正式全面可用。AWS 是第一家提供 Anthropic 原生平台体验的云服务商。

具体意味着什么？你的公司如果已经在用 AWS，现在可以直接通过 AWS 账号访问 Claude 的完整 API 能力——Messages API、web search、MCP connector、代码执行、文件上传都在内。不需要单独去 Anthropic 官网注册，不需要维护一套独立的账单和权限。

🔑
认证走 AWS IAM，不用额外管理 API key
💳
用量通过 AWS Marketplace 按消费计费，和其他 AWS 服务统一查看成本
📋
操作记录进 AWS CloudTrail，合规审计和现有体系打通

⚠️ 一个细节：这条路上的数据由 Anthropic 处理，不在 AWS 安全边界内。有严格数据合规要求的行业（医疗、金融），这和在 Amazon Bedrock 上调 Claude 是两条不同的路。

Developer

开发者那头：多个 Agent 同时跑，终于有地方看了

同一天，Claude Code 推出 Agent View，目前是 Research Preview。

开发者同时跑多个 Claude Code 会话，之前的办法是开多个终端标签页、用 tmux 或者手动记录——本质上是靠人肉管理并行。Agent View 把这些会话压缩进一张列表，每一行显示当前状态、是否在等待用户输入、Claude 最后回复了什么、上次交互时间。

它解决的问题不是「让 AI 做更多」，而是「让人知道 AI 在做什么」。

Pro、Max、Team、Enterprise 和 Claude API 计划都可以用。
