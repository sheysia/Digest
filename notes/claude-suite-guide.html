Claude 不只是聊天
五层能力，你用到了第几层？

从对话到设计，一个工具的五种打开方式
Issue #004 · May 2026

如果你正在读这篇文章，大概率你已经在用 Claude 了——问它问题、让它帮你改邮件、翻译一段话。

但很多人不知道的是，聊天只是 Claude 最基础的一层。往上还有四层能力，每一层都能让你做到之前做不到、或者要花很多时间才能做到的事。

不是说你必须全部用上。但知道有什么、适合什么场景，能帮你在需要的时候少走弯路。

Chat → Project → Cowork → Code → Design
第一层
从 Chat 到 Project — 让 Claude 记住你是谁
大多数人卡在这一步

Chat 就是最基本的一问一答。你问，它答，结束。

但你有没有遇到过这种情况——每次开新对话，都要重新解释一遍：「我是做数据的」「我在 DMV」「我的团队用 Tableau」「我上次跟你说的那个项目……」

Project 解决的就是这个问题。

你可以把 Project 理解成一个「带记忆的工作空间」。给它设一个系统指令（比如「你是一个面向非技术受众的 AI 科普编辑」），上传你的参考资料——品牌手册、过往文章、数据字典——之后在这个空间里的每次对话，Claude 都自带这些上下文。不需要每次从零开始。

写周报：建一个「工作汇报」Project，上传 KPI 模板和上月报告。之后每次直接说「根据这周的数据帮我更新」——它已经知道你的格式、口吻、数据结构。

管理长期项目：无论是 side project、备考、还是搬家规划——建一个 Project 把资料丢进去，每次聊天接着上次来。

内容创作：写作风格指南、爆款素材、选题库全放进去。要写东西时 Claude 直接按你的调性出稿，不用反复调教。

起步不用想太多。先建三个 Project——工作、写作、学习各一个。每个放一份最常用的参考资料，写一两句系统指令。用上一周你就会发现差别。
第二层
Cowork — 从「问答」变成「一起干活」
目前最被低估的功能

Chat 和 Project 的交互还是「你说一句、它回一段」。写好之后你得自己去复制、粘贴、排版。

Cowork 不一样。左边是对话窗口，右边是实时编辑的文档区域。你一边说需求，它一边在右边写。看到不满意的段落，直接框选说「这段改一下，太正式了」——它当场改。

整个过程就像坐在一个写手旁边，你指着屏幕说「这里改改那里调调」。文件始终在你面前。

写长文：在 Chat 里写长文来回复制粘贴效率太低。Cowork 里 Claude 直接在文档区写，你看着它写、随时打断。

整理资料：笔记、会议记录、参考链接——丢进去让它归类、提炼、生成结构化文档。

写方案：品牌指南和过往作品放进 Cowork，让 Claude 基于这些素材按你的风格产出新内容。

一句话区分——Chat 适合「问问题」，Cowork 适合「产出一个东西」。如果你最终需要一份可以直接用的文件，Cowork 是更好的选择。
第三层
Code — 不会写代码也能建网站
听起来吓人，门槛比你想的低得多

如果你的第一反应是「Code 跟我没关系」——停一下。

Claude Code 能做的远不止写代码。你可以用一句话让它做一个官网，把 Excel 变成可视化看板，做一个互动展示页，或者生成一个自动收集数据的问卷。所有这些，你不需要写一行代码——用中文描述你想要的就行。

在 Claude 桌面端点顶部 Code 标签，选 Opus 模型。

用自然语言描述你想要的——「帮我做一个简洁的个人品牌网站，首页大标题、个人介绍、服务列表、联系方式。」

看它自动生成，打开看效果。不满意就继续说：「标题再大一点，背景换浅灰。」

这个「说 → 看 → 改」的循环，就是所谓的 Vibe Coding。你不写代码，只描述想要的结果。

一个真实的例子：你正在读的这份 AI Digest 网页，就是这样做的。没有请开发者，没有买模板，核心时间不到两小时。

进阶技巧：让 Claude 在项目根目录创建一个 CLAUDE.md 文件，把所有项目信息写进去。下次打开它会先读这个文件，自动恢复上下文——字体、颜色、文件结构，不用从头解释。
第四层
Design — 不用 Canva 也能出视觉
从「能用」到「能看」的那一步

很多人不知道，Claude 可以直接生成带设计的内容——不是那种纯文字输出，而是排好版、配好色、可以直接用的页面和文档。

你在对话里说「帮我做一张信息图」「做一个项目进度看板」「设计一个活动邀请页」，它会直接在聊天窗口里渲染出一个可交互的成品。配色、字体、布局，全都帮你做好。

这不是 Figma，也不是 Canva，但对于大多数日常需求已经够用了。

数据可视化：把一份 CSV 或 Excel 丢给它，说「帮我做一组图表，展示季度趋势」——它直接生成带标注的交互式图表，比在 Excel 里调半天快得多。

演示和提案：「帮我做一个单页的项目提案，包含问题、方案、时间线」——它会输出一个排版好的页面，配色专业，可以直接截图发给团队。

社交媒体素材：文字排版、信息卡片、对比图、流程图——不需要打开任何设计软件，描述你要什么，它当场生成。

关键区分：Code 是建一个独立运行的东西（网站、工具），Design 更像是在对话中随手产出一个视觉成品。两者可以结合——先在 Chat 里用 Design 快速验证想法，满意了再用 Code 做成正式站点。

大多数人只用到了第一层。如果你现在还只是在 Chat 里问问题，试试建一个 Project，把最常用的上下文放进去。体验一下有记忆和没记忆之间的差距。

然后看看 Cowork，尤其是如果你的工作涉及大量写作和文档。

至于 Code——试着多走一步。只靠和 Chat 截图对话就能装上它，然后发现聊熟的老朋友就在那里，同样亲切，更加强大。

进阶
给 Code 装上 Skills — 让它先拷问你再动手

用了一段时间 Code 之后你可能会发现一个问题：它太听话了。你说什么它就做什么，但有时候你自己也没想清楚要什么——做到一半才发现方向不对，又得推翻重来。

这时候可以给它装一个叫 grill-me 的 Skill。装上之后，在 Code 里输入 /grill-me，Claude 会反过来拷问你：你到底要解决什么问题？用户是谁？边界条件考虑了吗？——一路追问到你答不上来为止。

听起来有点烦，但这恰恰是最省时间的一步。被 AI 怼一顿之后再动手，出来的东西完成度高很多。

Skill 是 Claude Code 的「插件」，由社区开发者共享。安装只需要一行命令。除了 grill-me，还有几个值得了解的：

git-guardrails — 给 Code 加一道安全网，拦截所有危险的 git 操作（比如强制推送、删除分支），防止 AI 误删你的代码。装上睡觉踏实。

to-prd — 和 Claude 聊完一个想法之后，它自动把对话整理成一份产品需求文档，直接提交到 GitHub。不用再手动总结。

这些 Skills 来自开发者 Matt Pocock 的开源项目，是目前 Claude Code 生态里最主流的一套工具集。你不需要懂代码就能用——在 Code 里输入安装命令，等它跑完就行。

源于生活，立足应用。
