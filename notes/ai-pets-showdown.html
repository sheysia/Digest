两家巨头不约而同
给自己的 AI 养了宠物

Anthropic 出了 Buddies，OpenAI 跟了 Codex Pets。认真的？
Issue #004 · May 2026 · 行业趣闻

你没看错。

就在 AI 军备竞赛打得最激烈的 2026 年五月，两家最严肃的 AI 公司——一家主打「AI 安全」，一家号称要做「通用人工智能」——几乎前后脚做了同一件事：给自己的编程工具加了一只电子宠物。

发生了什么

四月初，Anthropic 在 Claude Code 里悄悄上线了一个叫 Buddies 的功能——一个浮动的小动画角色，在你用 Claude Code 写代码的时候待在屏幕角落里，告诉你当前任务的状态：在跑、等你输入、还是已经完成。

五月第一周，OpenAI 跟进了。在 Codex 里推出了 Codex Pets——功能几乎一模一样：浮动小动画，实时显示任务状态，可以自定义角色。内置了猫、狗等 8 种预设，还能用 AI 生成你自己的宠物。

社区反应很快。OpenAI 那边，开发者们已经生成了各种自定义宠物——有人做了龙猫，有人做了 Clippy 复活版，有人做了迷你版 Sam Altman，甚至还有人做了 Anthropic CEO Dario Amodei 的像素版。

为什么值得关注

表面上看这就是两家公司在做可爱的小功能。但往深了想，它反映了一个趋势：编程工具开始竞争「体验」，而不只是「性能」。

AI 编程工具有一个天然的 UX 问题：它们在后台异步工作——跑任务、执行命令、迭代代码——而你在做别的事情。没有状态提示的话，你要么忽略它，要么不停切窗口检查。

宠物本质上是一个「可爱化的状态指示器」。它解决了一个真实的交互问题，只不过用了一种让人会心一笑的方式。

有人觉得这是 AI 工具开始不务正业，有人觉得这才是产品成熟的标志——当基础能力拉不开太大差距的时候，谁让用户用得更舒服、更愿意留下来，谁就赢了。

2026 年的 AI 编程工具，不再比谁跑分更高，开始比谁的吉祥物更萌。这个画面本身就已经很有趣了。

有一个被埋在可爱宠物底下的细节：OpenAI 在同一次更新中，让 Codex 可以自动检测和导入 CLAUDE.md 配置文件——也就是说，Claude Code 用户可以两步迁移到 Codex。宠物是门面，配置导入才是武器。

源于生活，立足应用。偶尔也看看热闹。
