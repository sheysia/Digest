Anthropic 发了十个金融 Agent
金融之外的人也可以看看

一次行业发布，但里面藏着 Agent 落地的通用思路
Issue #004 · May 5, 2026 · 快速解读

今天 Anthropic 发了一篇公告，标题很朴素：「Agents for Financial Services」。信息量不小，简单帮你梳理一下。

三件事同时发布：

第一，十个即插即用的金融 Agent 模板。覆盖 pitchbook 制作、KYC 筛查、月末结账、估值审核、财务对账、盈利分析等——全部是金融行业里最耗时的工作。每个模板可以作为 Cowork/Code 插件运行，也可以作为 Managed Agent 在平台上自主运行。

第二，Claude 正式入驻 Microsoft 全家桶。Excel、PowerPoint、Word 已上线插件，Outlook 即将跟进。关键是：上下文跨应用打通——在 Excel 里建的模型，移到 PowerPoint 做 deck 时不用重新解释。

第三，数据生态扩展。新增了 Dun & Bradstreet、Moody's、FactSet、S&P Capital IQ 等金融数据连接器，Claude 可以调用这些平台的实时数据来辅助任务。

金融之外可以看什么

这次发布表面上是一个行业方案，但里面有一些思路可能适用范围更广。

之前大家讨论 AI Agent，多半停留在概念层面——「它可以帮你做事」「它可以自主执行」。具体长什么样、怎么用、跟现有工具怎么配合，一直比较模糊。

这次给出了一个相对具体的参考：

Agent = 技能 + 数据连接 + 子任务分工。每个模板包含三样东西：Skills（领域知识和指令）、Connectors（接入真实数据源）、Sub-agents（把大任务拆成小任务分给不同的 Claude 执行）。

这个架构本身不限于金融。类似的模式或许可以用在医疗（病历审核）、教育（课件生成）、内容运营（每日摘要）——技能和数据源换了，骨架可能是通用的。

Claude 进 Excel，值得留意

Claude for Excel 看起来只是「在表格里用 AI」，但可以多想一步：金融行业里大量的定量工作——投行建模、基金估值、保险精算、企业财报——最终都落在 Excel 里。

AI 直接嵌入这个场景，意味着它不只是在旁边给建议，而是可以在你的工作文件里帮你审公式、跑敏感度分析、从财报建模。做完之后还能带着上下文移到 PowerPoint 出 deck，不用重新解释一遍。

从公告里的客户引用来看，Citadel 已经在用 Claude for Excel，Walleye Capital 提到他们 400 人的基金里所有员工都在用 Claude Code——不只是技术团队。当然，这些是官方引用，实际使用深度可能有差异，但方向可以参考。

值得留意的信号

Opus 4.7 露面了。公告里提到这些 Agent 建议配合 Claude Opus 4.7 使用，在 Vals AI 的金融 Agent 基准测试上拿到 64.37%。看起来 4.6 之后的新模型已经在特定场景里落地了。

Managed Agent 值得关注。除了作为插件在桌面上跑，这些 Agent 还能以「托管模式」在 Claude Platform 上自主运行——处理几个小时的长任务、按计划每晚自动执行，配有权限管理、凭证库和审计日志。如果这个模式成熟，可能会模糊「AI 助手」和「AI 员工」之间的界限。

数据合规被放在很重要的位置。Anthropic 反复提到「governed access」——每个数据连接器都有权限控制，每步操作都有审计记录。对金融机构来说，可追溯性可能比模型能力本身更关键。

和本期那篇趋势信号呼应一下——AI Agent 正在从概念往产品走。今天这个发布算是一个具体的落地样本。金融是第一个吃螃蟹的行业，但背后的架构思路可能是通用的，后续其他行业也许会出现类似的模板化方案。建议保持关注，但不必着急。
