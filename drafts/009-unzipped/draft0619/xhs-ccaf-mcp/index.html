01

MCP 是什么

USB-C 类比
USB-C 为电子设备提供统一接口，MCP 为 AI 应用提供统一的外部系统连接标准。

MCP 把工具定义和执行的负担从你的服务器转移到专门的 MCP Server。

没有 MCP：你需要自己写 GitHub 所有功能的 tool schema + 调用函数。

有了 MCP：GitHub MCP Server 已经封装好一切，你只需连接。

MCP ≠ Tool Use
MCP Server 提供已定义好的工具，Tool Use 是 Claude 调用工具的机制。两者互补，不是同一件事。

用 Python SDK 定义工具

from mcp.server.fastmcp import FastMCP mcp = FastMCP("DocumentMCP") @mcp.tool(   name="read_doc_contents",   description="Read a document..." ) def read_document(   doc_id: str = Field(     description="Id of the document"   ) ):   return docs[doc_id]

不需要手写 JSON schema，装饰器 + 类型提示自动生成

Field(description=...) 告诉 Claude 每个参数的用途

异常处理直接用 Python raise ValueError

Resources 与 Prompts

Resources：暴露只读数据
类型URI说明

用户通过 /命令 触发，返回精心编写的 prompt 消息列表

适合封装反复使用的复杂提示词，如文档格式化、代码审查

06

三大原语：谁在控制？

原语控制者触发方式典型场景
Tools模型 (Claude)Claude 自主决定调用搜索、计算、API 调用
Resources应用代码App 决定何时获取自动补全、上下文注入
Prompts用户用户点击 / 输入命令格式化、审查、报告
考试判断框架
给 Claude 新能力 → Tools
获取数据给 UI 或上下文 → Resources
创建用户可触发的工作流 → Prompts

MCP Inspector：mcp dev server.py 启动浏览器调试界面，测试所有三种原语
