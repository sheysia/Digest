关于这门课

Anthropic 官方认证课程，共 6 节，教你如何让 Claude Code 学一次，自动用。

涵盖 Skills 的创建、配置、多文件组织、与 CLAUDE.md / Hooks / Subagents 的区别、团队共享与排错。

In this course, you'll learn how to stop repeating yourself and start teaching Claude once.

You'll discover what Skills are, how they differ from CLAUDE.md, hooks, and subagents. You'll create Skills from scratch, configure advanced options like allowed-tools, organize multi-file structures with progressive disclosure, share them via Git / plugins / enterprise settings, and troubleshoot common issues.

Skills 是什么 + 在哪里

Skills = 可复用的 markdown 指令文件，Claude Code 根据任务自动匹配加载。
存放位置
位置路径共享
个人~/.claude/skills/仅自己，跨项目
项目.claude/skills/Git 共享团队
优先级（考试高频）
Enterprise > Personal > Project > Plugins
02

核心结构与加载机制

--- name: pr-review description: Reviews pull requests              for code quality. --- （下面写具体指令）
加载机制

启动时只加载 name + description，不占满 context

请求匹配时才加载完整内容

加载前会有确认提示

description 是 Claude 匹配请求的唯一依据

03

Frontmatter 与工具限制

字段必填作用
name✅标识 skill
description✅Claude 匹配请求的依据
allowed-tools限制 skill 可用的工具
model指定使用哪个模型
allowed-tools: Read, Grep, Glob, Bash # 只允许读操作 → 安全审计/只读场景

SKILL.md 控制在 500 行以内

详细参考放 references/ 子目录，按需读取

脚本执行不加载源码，只输出消耗 token

04

Skills vs 其他功能

功能加载时机触发方式隔离
CLAUDE.md每次对话自动_
Skills按需匹配自动_
Slash Cmd不自动手动输入_
Hooks事件触发自动_
Subagents独立上下文委派任务✅
考试判断框架
始终生效 → CLAUDE.md
按任务自动匹配 → Skills
手动调用 → Slash Commands
确定性合规 → Hooks
隔离执行 → Subagents

共享与分发

Subagent × Skills 陷阱（考试常考）
❌ Subagent 不自动继承 skills
❌ 内置 agent（Explorer/Plan/Verify）完全无法访问 skills
✅ 自定义 subagent 在 frontmatter 里显式声明 skills 字段才能使用
