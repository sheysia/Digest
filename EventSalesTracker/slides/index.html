Event Sales Tracker
用 AI 一个下午做一个工具

线下活动销售记录 PWA：从需求到上线的完整复盘

Syneira LabDMVoice AI Group · sale.syneira.org
#PWA #React #offline-first

// why

起因：社区活动需要工具支持

社区有一场线下活动（yard sale / pop-up market），我们社群作为 support，需要现场记录销售。

场景约束

单人单 iPad，户外使用，网络不稳定，一边聊天一边记账

现有工具不合适

POS 系统太重，记事本太轻，Excel 不够快

决定自己做

最小可用
当天能上线
扫码就能用
不需要下载安装

// tech stack

技术选型：能不建服务器就不建

React + Vite + TS

开发速度快，静态构建，无 SSR

IndexedDB (Dexie)

数据存设备本地，从不上传，浏览器就是数据库

Tailwind CSS v4

@theme token 出 UI，触屏优先，大按钮

SheetJS

浏览器端解析 Excel，不需要服务器

vite-plugin-pwa

Workbox 离线缓存，添加到主屏幕后完全离线可用

Vercel

push 即部署，自定义域名 sale.syneira.org

纯前端 PWA：零后端、零运维、零登录、零成本

// design decisions

五个关键设计决策

决策 1两张表（Items + Sales）→ 一个商品多次卖出、不同价格、不同收款方式，撤销干净
决策 2Sold 弹窗两次点击完成 → 户外单手操作，成交价和 Cash 都默认填好
决策 3按收款方式分项汇总 → Cash / Venmo / Zelle 对账才是真正需求
决策 4强推 Add to Home Screen → Safari 7 天清除策略，主屏幕 PWA 豁免
决策 5默认价格 $1 → yard sale 物品金额小，减少输入

// user feedback → ship

迭代：反馈驱动，几分钟上线

v1.0基础功能上线：导入、售出、汇总、导出
反馈 1价格默认太高 → 改为默认 $1
反馈 2希望能拍照识别商品 → 加了可选拍照，缩略图显示在卡片上
反馈 3需要双域名 → 短域名方便传播 + 长域名保持透明度
反馈 4需要现场指引 → 生成可打印 QR 指引卡

每次改动都是 AI 辅助 → git push → Vercel 自动部署 → 刷新即生效

// privacy & trust

隐私与信任设计

数据完全本地

所有销售数据存在设备上，从不上传，无服务器、无数据库

无登录无追踪

不需要注册账号，不收集任何用户信息

透明声明

Settings 底部明确说明技术架构和隐私策略

双域名策略

sale.syneira.org
短域名，方便记忆和口头传播

eventsale.vercel.app
Vercel 托管地址，保持平台透明度

"Sales tracker for your convenience, optional to use"

// takeaways

给社群的启发

不是所有项目都需要后端

PWA + IndexedDB 在临时活动场景下完全够用，零成本零运维

AI 辅助 = 速度

一个下午从零到上线，AI 处理样板代码，人专注决策和体验

反馈 → 迭代 → 上线

每次改动几分钟 push 上线，这个节奏比完美设计更重要

最小可用 > 完美设计

先做能用的，再根据真实反馈优化，不猜用户需要什么

信任靠透明

数据本地、隐私声明、正规托管，用户看到就放心
